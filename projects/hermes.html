Hermes Beta

The iOS command deck for my personal agent infrastructure — the operations layer that lets one person run a nine-product portfolio from a phone.

The Problem

Running nine products solo generates a very particular kind of noise: support email, outreach pipelines, brand mentions, agent conversations, app statuses. Checking all of it means sitting at a desk cycling through dashboards — which is exactly the kind of recurring, low-judgment work that should belong to software.

What I Built

Hermes is the iOS client for a self-hosted agent stack that already runs the portfolio's operations on cron. The app puts that infrastructure in a pocket:

  • Email triage from anywhere Gmail triage over OAuth — the agent drafts, archives, and flags; I approve from the phone.
  • One briefing instead of five dashboards Every morning at 8:00: email summary, overnight VibeCS outreach results, Reddit brand mentions, calendar, and per-product status — aggregated into a single readable digest.
  • Ask the vault Natural-language queries against my Obsidian knowledge base — "what did we learn from BlockRate?" gets an answer with sources, on the train.
  • Spot-check the agents Live views into Send Remy agent conversations and SavvyCash debugging — the portfolio's other AI systems, supervised from one app.

The meta-point is the thesis eating its own tail: I run the portfolio through an agent I built. Hermes is the context layer for being me.

Stack
iOS
Swift 6, SwiftUI
Agent layer
Self-hosted automation stack on cron — triage, monitoring, briefings
Integrations
Gmail API (OAuth), Supabase, Reddit API, Obsidian vault
Status & Outcome

Private beta — running daily on my own device, which is both its distribution and its product review. It ships publicly if and when the agent stack generalizes beyond an audience of one.
